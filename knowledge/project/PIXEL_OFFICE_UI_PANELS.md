# PIXEL OFFICE UI PANELS

## Purpose
This file defines the conceptual UI panels for the Pixel Office interface.

## Main Principle
The office view should be complemented by clear side panels and inspectors.

## Core UI Panels

### 1. Office View Panel
- show the pixel office itself
- show rooms
- show agents
- show movement and status indicators

### 2. Agent Inspector Panel
- role
- current status
- current task
- last event
- recent outputs
- dependency state
- error notes

### 3. Task Queue Panel
- task title
- assigned agent
- priority
- status
- waiting conditions

### 4. Event Feed Panel
- task assigned
- status changed
- approval requested
- error raised
- completion events

### 5. Approval Panel
- content approval
- proposal approval
- sensitive legal response approval
- deployment confirmation

### 6. Alerts Panel
- agent error
- blocked dependency
- failed integration
- overdue follow-up
- legal/regulatory alert

### 7. Communications Panel
- incoming commands
- outgoing summaries
- approval requests
- notifications

### 8. Office Summary Panel
- number of agents online
- active tasks
- waiting approvals
- blocked items
- recent completions

## Final Principle
The Pixel Office UI should combine visual presence with operational control and inspection.
