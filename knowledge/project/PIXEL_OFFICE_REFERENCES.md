# PIXEL OFFICE REFERENCES

## Purpose
This file collects external references related to Pixel Office, visual agent teams, and office-style multi-agent interfaces.

Its purpose is to document which real-world projects and discussions inspire the Formula Finance AI Office visual layer.

## Main Principle
These references are not meant to be copied blindly.

They should be used as inspiration for:
- interface ideas;
- architecture decisions;
- event-driven visualization patterns;
- room-based office design;
- agent observability concepts.

## Reference 1 — Pixel Agents
Link:
https://github.com/pablodelucca/pixel-agents

Why it matters:
Pixel Agents is one of the clearest examples of a pixel-office style visualization for AI agents.

What is useful for this project:
- agents shown as visible pixel characters;
- desk / room logic;
- observable state;
- agent activity reflected in animation;
- idea of a visual control room for multi-agent work.

Important note:
Pixel Agents observes Claude Code transcript/state behavior rather than requiring deep modification of agent logic.

## How To Use These References
Use these references for:
- visual inspiration;
- room layout ideas;
- status visualization ideas;
- event-driven architecture ideas;
- technology stack comparison.

Do not assume:
- their exact architecture is automatically right for Formula Finance;
- their priorities match a consulting/business operations use case.

## Final Principle
Pixel Office for Formula Finance should be inspired by real working references, but adapted for business operations, Telegram workflows, staged deployment, and beginner-friendly implementation.
