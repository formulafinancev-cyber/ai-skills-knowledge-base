# TEMPLATE MEETING NOTE

## Purpose
This file defines the standard structure for meeting notes inside the Formula Finance project.

Its purpose is to make meeting outputs easier to capture, easier to review, and easier to turn into next steps.

## Main Principle
A meeting note should not be a loose dump of phrases.

It should help answer:
- what was discussed;
- what matters;
- what decisions were made;
- what remains unclear;
- what happens next.

## Standard Meeting Note Structure

### 1. Meeting Title
A short and clear title for the meeting.

Example logic:
- client meeting;
- internal planning meeting;
- proposal review;
- KPI discussion;
- project coordination call.

### 2. Date
Record the meeting date clearly.

### 3. Participants
List the main participants.

This can include:
- internal roles;
- client representatives;
- decision makers;
- observers if relevant.

### 4. Context
Briefly describe why the meeting happened.

Examples:
- initial discovery call;
- follow-up after proposal;
- internal reporting discussion;
- CRM review;
- analytics clarification.

### 5. Main Topics Discussed
List the main topics covered during the meeting.

This section should be:
- concise;
- structured;
- easy to scan.

### 6. Key Findings
Capture the most important points that emerged.

Examples:
- pain points;
- business priorities;
- reporting issues;
- expectations;
- blockers;
- needs that became clearer during the conversation.

### 7. Decisions Made
List any clear decisions made during the meeting.

Examples:
- proposal needed;
- next call scheduled;
- scope to be clarified;
- reporting direction accepted;
- diagnostic phase approved.

### 8. Open Questions
List any unresolved questions that still need clarification.

Examples:
- missing client data;
- scope uncertainty;
- role uncertainty;
- pricing detail still pending;
- unclear implementation timeline.

### 9. Next Steps
List concrete next actions.

Each next step should ideally show:
- action;
- responsible side or role;
- expected direction.

### 10. Notes
Optional extra notes that may be useful later.

This section should not become the main body of the document.  
It is only for supporting context.

## Quality Rules
A meeting note should be:
- clear;
- structured;
- short enough to scan;
- detailed enough to be useful later.

It should avoid:
- chaotic raw transcription;
- long unstructured paragraphs;
- mixing decisions and guesses;
- forgetting next steps.

## Role Use
This template is useful for:
- Colleague;
- Coordinator;
- CRM Agent;
- Proposal Agent;
- Analyst.

It can be used both for client-facing and internal meetings.

## Future Expansion
This file can later be expanded with:
- short-form meeting note variant;
- internal meeting variant;
- client meeting variant;
- action-focused meeting note variant.

## Final Principle
A meeting note should help preserve useful context and make action easier after the meeting ends.
