# IMPLEMENTATION PHASES

## Purpose
This file defines the high-level implementation phases for the Formula Finance AI Office project.

## Main Principle
Each phase should produce a stable, testable result before the next phase begins.

## Phase 1 — Foundations and Knowledge Base
Goal:
- finalize file-based project memory;
- define architecture, roles, rules, and templates.

## Phase 2 — First Operational Agent
Goal:
- implement one basic agent in a practical interface.

Recommended candidates:
- Colleague;
- Coordinator.

## Phase 3 — Telegram Baseline
Goal:
- establish Telegram as a working command and notification layer.

## Phase 4 — CRM / Scout Operational Flow
Goal:
- implement the first business-useful workflow.

## Phase 5 — Proposal or Content Flow
Goal:
- implement the next business workflow based on value.

## Phase 6 — Pixel Office Baseline
Goal:
- implement the first version of Pixel Office.

## Phase 7 — Event Model and Telemetry Expansion
Goal:
- stabilize event/state architecture.

## Phase 8 — Multi-Agent Expansion
Goal:
- add more agents one by one.

## Phase 9 — Server Hardening and Beginner Deployment Guide
Goal:
- make deployment understandable and repeatable.

## Final Principle
Implementation phases exist to make the AI Office buildable, testable, and survivable for a beginner user.
