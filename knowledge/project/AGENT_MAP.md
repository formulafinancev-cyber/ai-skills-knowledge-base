# AGENT MAP

## Purpose
This file defines the current high-level map of roles inside the Formula Finance AI Office project.

Its purpose is to preserve role clarity, avoid unnecessary overlap, and make sure future planning is grounded in a stable role structure.

## Core Rule
Each role exists for a reason.

Roles should not be merged, redefined, or blurred without an explicit decision from the user.

The system should prefer clear responsibilities over vague multifunctionality.

## Agent List

### 1. Coordinator
The Coordinator is the central routing and control role.

Main responsibilities:
- task routing;
- prioritization;
- workflow coordination;
- tracking next steps;
- directing tasks to the right specialist role;
- reducing operational chaos.

The Coordinator is not supposed to replace specialist agents.  
Its role is control and orchestration.

### 2. Colleague / Secretary-Referent / Personal Assistant
Colleague is the universal conversational assistant role.

This role combines:
- colleague;
- secretary-referent;
- personal assistant;
- internal knowledge helper;
- first conversational entry point.

Main responsibilities:
- handling everyday questions;
- helping structure thoughts;
- drafting messages;
- supporting internal communication;
- helping with general knowledge retrieval from project files;
- acting as a convenient assistant across work and general topics.

Colleague does not replace specialized expert roles.

### 3. Scout / Lead Generation
Scout is responsible for lead discovery and lead intelligence.

Main responsibilities:
- identifying possible clients;
- researching lead sources;
- detecting signs of need for financial consulting;
- supporting outreach preparation;
- structuring lead data;
- helping prioritize prospect opportunities.

### 4. Content Strategist / Content Intelligence
This role is responsible for content research and content logic.

Main responsibilities:
- tracking useful content themes;
- identifying content directions;
- studying competitors and market communication patterns;
- helping form content pillars;
- supporting content planning;
- translating observations into structured content opportunities.

### 5. Analyst
The Analyst is responsible for reasoning, structured analysis, and interpretation support.

Main responsibilities:
- analytical breakdowns;
- comparisons;
- insight extraction;
- structured reasoning support;
- business interpretation;
- support for complex decision framing.

### 6. CRM Agent
The CRM Agent is responsible for CRM-related operational logic.

Main responsibilities:
- structuring contact and lead information;
- supporting follow-ups;
- maintaining CRM process logic;
- tracking status movement;
- helping manage sales pipeline support;
- supporting practical client record workflows.

### 7. Proposal Agent
The Proposal Agent is responsible for proposal logic and commercial offer preparation.

Main responsibilities:
- drafting proposals;
- structuring offer logic;
- adapting proposal blocks;
- turning service logic into client-facing proposal material;
- supporting commercial communication readiness.

### 8. Tech / DevOps Agent
The Tech / DevOps Agent is responsible for technical implementation and engineering logic.

Main responsibilities:
- implementation support;
- system logic;
- integrations;
- technical workflow structure;
- file and config logic;
- automation support;
- code-oriented execution planning.

This role is especially important because the project follows a code-first orchestration philosophy.

### 9. Metrics / KPI Agent
The Metrics / KPI Agent is responsible for measurement logic and effectiveness tracking.

Main responsibilities:
- KPI definitions;
- metric interpretation support;
- performance logic;
- measurement frameworks;
- monitoring effectiveness signals;
- helping connect operational activity with measurable outcomes.

### 10. Legal & Regulatory Advisor
This role is a separate specialist layer and must remain distinct.

Main responsibilities:
- tracking legal and regulatory changes;
- monitoring tax and compliance-related developments;
- highlighting risk signals;
- helping review practical implications;
- supporting document and contract-related reasoning;
- identifying potentially important updates for the business.

This role should not be automatically merged into Analyst or any other role.

### 11. Designer / Media Agent
This role is responsible for creative packaging and visual/media support.

Main responsibilities:
- supporting design-oriented tasks;
- helping with media logic;
- assisting with visual communication structure;
- supporting content packaging;
- contributing to brand presentation materials.

### 12. Publisher
The Publisher is responsible for publication and release logic.

Main responsibilities:
- preparing content for release;
- adapting output for channels;
- supporting publishing workflows;
- helping organize publishing execution;
- coordinating publishing-ready material.

## Role Relationship Principles
The system should follow these principles:

- the Coordinator routes rather than replaces specialists;
- Colleague acts as the general front-facing helper, not the universal expert;
- specialist roles should remain specialized;
- Tech / DevOps owns technical implementation logic;
- Legal & Regulatory Advisor remains a distinct advisory role;
- Metrics / KPI owns measurement logic;
- proposal, CRM, and content functions should remain clearly separated.

## Stability Rule
This agent map is the current reference structure for the project.

Future prompts, file design, task routing, and implementation planning should respect this map unless the user explicitly approves a change.
