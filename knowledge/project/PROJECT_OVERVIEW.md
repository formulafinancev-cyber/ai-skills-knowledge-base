# PROJECT OVERVIEW

## Project Name
Formula Finance AI Office

## Project Type
Multi-agent AI operating system for Formula Finance.

## Purpose
This project is being built to create a structured AI office for Formula Finance.  
Its purpose is to support financial consulting, internal operations, analytics, lead generation, CRM support, proposal preparation, content operations, publishing workflows, legal and regulatory monitoring, and internal assistance.

## Core Idea
The system is designed as one unified Claude Code-based environment with multiple specialized agent roles, a shared knowledge base, shared operating rules, and clear task routing.

This is not a collection of unrelated bots.  
It is a single operational system with role separation, common project logic, and phased implementation.

## Main Interaction Layer
The primary interaction layer for the owner and the team is Telegram.

## Implementation Philosophy
The system must be built step by step.

The project should first establish:
1. project structure;
2. shared rules;
3. knowledge base;
4. agent roles;
5. workflows;
6. implementation sequencing.

Code should not come first.  
Structure, clarity, and durable project organization come first.

## Model Strategy
The main runtime model for daily work is Claude Sonnet 4.6.

Claude Opus 4.7 and OpenAI Codex are both used for engineering work, code writing, implementation planning, refactoring, and technical development.  
The user manually combines Opus and Codex depending on the task.

## Long-Term Goal
The long-term goal is to build a clean, scalable, maintainable AI office for Formula Finance, where each role has clear responsibilities, project knowledge is organized, and implementation can progress in an orderly way without chaos.

## Important Constraint
The system is intended to be code-first in orchestration.  
n8n is not used as the main orchestration layer.

## Working Principle
All future work inside this project should support:
- better structure;
- clearer documentation;
- stronger role separation;
- practical implementation readiness;
- reduced chaos;
- long-term usability.
