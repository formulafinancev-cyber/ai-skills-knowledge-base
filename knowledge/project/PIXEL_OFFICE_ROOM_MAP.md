# PIXEL OFFICE ROOM MAP

## Purpose
This file defines the conceptual room map for the Pixel Office in the Formula Finance AI Office project.

Its purpose is to organize the visual office around functional business zones rather than random decoration.

## Main Principle
Rooms should reflect work functions.

A room is not just a visual location.  
It helps users understand what type of work is happening there.

## Proposed Rooms and Zones

### 1. Reception / Entry Zone
Purpose:
- visual entry into the office;
- high-level overview feeling;
- possible home screen area.

Possible role:
- welcome state;
- global office summary.

### 2. Coordinator Desk / Operations Zone
Purpose:
- central coordination;
- routing awareness;
- overview of what is active or blocked.

Mapped roles:
- Coordinator;
- possibly Colleague in support mode.

### 3. Sales / CRM Room
Purpose:
- lead generation;
- CRM updates;
- opportunity progression;
- follow-up logic.

Mapped roles:
- Scout;
- CRM Agent;
- Proposal Agent (partly connected).

### 4. Proposal / Meeting Room
Purpose:
- proposal preparation;
- commercial review;
- meeting preparation;
- client-facing package assembly.

Mapped roles:
- Proposal Agent;
- Coordinator;
- Colleague support.

### 5. Analytics Desk
Purpose:
- KPI work;
- reporting logic;
- interpretation;
- analysis.

Mapped roles:
- Analyst;
- Metrics / KPI Agent.

### 6. Legal / Regulatory Desk
Purpose:
- legal and regulatory monitoring;
- alerts;
- source review;
- interpretation support.

Mapped roles:
- Legal & Regulatory Advisor.

### 7. Content Studio
Purpose:
- content planning;
- content idea work;
- writing support;
- publishing preparation.

Mapped roles:
- Content Strategist;
- Publisher;
- media-support roles if added later.

### 8. Server / Automation Room
Purpose:
- integrations;
- API jobs;
- automation processes;
- background tasks;
- infrastructure health.

Mapped roles:
- Tech / DevOps Agent;
- automation subsystems.

### 9. Meeting Room
Purpose:
- approvals;
- reviews;
- human decision points;
- coordination moments.

Possible use:
- agent needs confirmation;
- proposal review;
- content approval;
- issue escalation.

### 10. Break / Idle Zone
Purpose:
- visual place for idle agents;
- low-priority waiting state;
- no active task state.

This helps the office feel readable.

## Visual Logic Rule
Agents should appear in the room that best matches their current work context.

If an agent changes type of work, it may move rooms or show state changes accordingly.

## Future Expansion
The room map may later include:
- finance archive area;
- social wall / publishing wall;
- alert center;
- executive dashboard wall.

## Final Principle
The Pixel Office room map should make business functions visually understandable at a glance.
